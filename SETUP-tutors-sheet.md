# 講師検索ページ（tutors.html）とGoogleフォームの接続手順

講師応募フォームの回答を、AIが自動でタグ分けし、`tutors.html` の条件検索に反映します。
**個人情報を守るため、スプレッドシートは2つに分けて運用します。**

---

## 全体の流れ

```
講師が応募フォームに回答（自由記述でOK）
      ↓  回答スプレッドシート【非公開】に1行追加
      ↓  ★AIが自動で読み取り → タグ列・コメント・指導科目などを記入
      ↓  運営が中身を確認 → 【評価】を手入力・【公開】をTRUE
      ↓  メニュー「KKM → 公開用シートへ同期」
   公開用スプレッドシート【リンク閲覧可】に、公開OKな項目だけコピー
      ↓
   tutors.html が公開用シートを読み込んで表示
```

- **タグはAIが自動入力**。運営は「評価の入力」と「公開の切替」だけ。
- **評価は後から手入力**。評価4.0未満・公開OFFの間はサイトに出ません。

---

## 2つのスプレッドシート（ここが重要）

| | ① 回答シート | ② 公開用シート |
|---|---|---|
| 中身 | フォーム回答（氏名・メール含む）＋AIタグ＋評価 | 公開OKな項目だけ |
| 共有 | **非公開のまま**（自分だけ） | **リンクを知っている全員＝閲覧者** |
| サイトが読む | ✕ | **◯（これのIDを貼る）** |

> なぜ分ける？…サイトがデータを読むにはシートを「リンク公開」にする必要があり、
> 回答シートをそのまま公開すると**氏名・メール・学生証まで見えてしまう**ためです。

---

## 1. 回答シートに列を追加する

フォーム回答シートの右側の空き列に、次の見出しを1行目に追加します（AIと運営が使う列）。

`ニックネーム` `学部学年` `指導科目` `コメント` `出身地` `きょうだい` `幼少期`
`学校歴` `学習スタイル` `得意不得意` `関わり方` `コミュニケーション` `やる気スイッチ`
`AI処理日時` `評価` `対応サービス` `公開`

- `評価`『公開』… **運営が手入力**
  - `評価`：10基準の平均（4.0以上で掲載）
  - `公開`：掲載してよければ `TRUE`
- `対応サービス`… **応募者がフォームで選ぶ**（運営が上書きも可）
  - サイトは日本語をそのまま理解します。値：`家庭教師` / `教育・受験相談` /（両方チェックで）両方
  - `tutor` / `consult` / `both` でも可。**空欄なら「両対応」扱い**
  - ⚠️ この列は **1つだけ**にすること（後述のとおりフォーム質問の列名を `対応サービス` にする）
- `希望時給`… **応募フォームの回答をそのまま表示**（家庭教師の先生のカードにのみ表示）
  - フォームの「希望時給」列がそのまま使われます（列名 `希望時給` のまま）
  - 例：`2,000円` → カードに「希望時給 2,000円／時」と表示
- それ以外 … **AIが自動で記入**

#### 対応サービスをフォームで聞く方法（AIの変更は不要）
1. 応募フォームに **チェックボックス（複数選択可）** の質問を追加
   - 質問文の例：「対応できるサービス」／選択肢：**家庭教師**・**教育・受験相談**
2. 回答スプレッドシートで、その質問の**列の見出しを `対応サービス` に**しておく
   （見出しの文字は編集してOK。以降の回答も同じ列に入り続けます）
3. これだけで、家庭教師だけ／教育・受験相談だけ／両方（両方チェック）が
   そのままサイトに反映されます。**スクリプトもAIも変更不要**です。

### フォーム項目 → タグ の対応（AIが判断）

| タグ列 | 主な判断材料（フォーム項目） | 入る値 |
|---|---|---|
| 出身地 | 出身中学・高校、最寄り駅 | local / city / transfer |
| きょうだい | 兄弟構成 | first / middle / last / only |
| 幼少期 | ご自身の幼少期の過ごし方 | lessons / free |
| 学校歴 | 出身中学・高校、大学の入試種別 | public / jhs / hs（複数可） |
| 学習スタイル | 中高時代の勉強スタイル | self / cram |
| 得意不得意 | 性格・特性、勉強について | effort / inquiry |
| 関わり方 | 目指す家庭教師像、得意な教え方 | companion / leader |
| コミュニケーション | 性格・特性、得意な教え方 | empathy / logical |
| やる気スイッチ | 得意な教え方、目指す像 | competition / curiosity / safety（複数可） |
| コメント | 自己PR、性格、目指す像 | 語り口調の一言（AI生成・氏名なし） |
| ニックネーム | 氏名の「下の名前」 | 例）たく先輩（姓・フルネームは使わない） |
| 学部学年 | 学部＋学年 | 例）神戸大 工学部 2年（直接転記） |
| 指導科目 | 指導可能科目 | 例）数学・理科（直接転記） |

### 値コードの意味

- 出身地：`local`=地方出身 / `city`=都市部出身 / `transfer`=転勤族・引っ越し経験あり
- きょうだい：`first`=長子 / `middle`=中間子 / `last`=末っ子 / `only`=一人っ子
- 幼少期：`lessons`=習い事多め派 / `free`=のびのび派
- 学校歴：`public`=公立中心 / `jhs`=中学受験経験あり / `hs`=高校受験経験あり
- 学習スタイル：`self`=独学・工夫型 / `cram`=塾・予備校活用型
- 得意不得意：`effort`=努力型 / `inquiry`=探求型
- 関わり方：`companion`=伴走型 / `leader`=牽引型
- コミュニケーション：`empathy`=共感・ほめて伸ばす / `logical`=論理的・課題解決
- やる気スイッチ：`competition`=競争・達成感 / `curiosity`=好奇心・面白さ / `safety`=安心感

## 2. AI自動入力を設定する（Apps Script）

スクリプト本体は **`apps-script/tutors-classify.gs`** です。

1. 回答シートで「拡張機能 → Apps Script」を開き、`tutors-classify.gs` の中身を貼り付けて保存
2. [Google AI Studio](https://aistudio.google.com/app/apikey) でAPIキーを発行（無料枠あり）
3. Apps Scriptの「プロジェクトの設定 → スクリプト プロパティ」に
   `GEMINI_API_KEY` としてキーを登録
4. スクリプト上部 `CONFIG.PUBLIC_SPREADSHEET_ID` に、次で作る**公開用シートのID**を入れる
5. 関数 `installTrigger` を1回実行（権限を許可）
   → 以後、**フォーム送信のたびに自動でAI分類**されます
6. 既存の回答は「メニュー KKM → 選択行をAI分類」または「未処理をまとめてAI分類」

> Gemini以外のAI（Claude等）を使う場合は、スクリプト末尾の `callAI()` だけ差し替えればOK。

---

## 3. 公開用シートを作って共有する

1. **新しいスプレッドシートを作成**（名前は「KKM公開用」など）
2. URLの `/d/` と `/edit` の間が **ID**：
   `docs.google.com/spreadsheets/d/【ここがID】/edit`
   → このIDを、Apps Scriptの `CONFIG.PUBLIC_SPREADSHEET_ID` と、
     `tutors.html` の `CONFIG.SHEET_ID` の**両方**に貼る
3. 公開用シートを **「共有」→「リンクを知っている全員」→「閲覧者」** にする
   （※回答シートは共有しない＝非公開のまま）

```js
// tutors.html 上部
var CONFIG = { SHEET_ID: '公開用スプレッドシートのID', SHEET_GID: '0' };
```

---

## 4. 日々の運用（応募が来たら）

1. AIがタグを自動記入（`AI処理日時` が入る）
2. 回答の中身を確認し、**`評価` を手入力**（10基準の平均）
3. 掲載してよければ **`公開` を TRUE**
4. メニュー **「KKM → 公開用シートへ同期」** を実行
   → 評価4.0以上・公開TRUEの人だけがサイトに反映されます

---

## 仕組み（かんたん）

- サイトは**公開用シートだけ**を読み込みます（サーバー不要）
- 公開用シートには、氏名・メール等は**そもそも入っていません**（列の選抜）
- 評価4.0未満・公開OFFの応募者は同期されず、**サイトに出ません**（行の選抜）
- 家庭が選んだ条件と各先生のタグの一致数を数え、**多い順に並び替え**

## うまくいかないとき

- サイトが赤帯（読み込み失敗）→ 公開用シートの共有が「リンクを知っている全員／閲覧者」か確認
- 先生が出ない → `評価`が4.0以上か、`公開`がTRUEか、同期を実行したか確認
- 同期しても出ない → **見出しの列がダブっていないか確認**（`評価`や`ニックネーム`が2つあると、
  スクリプトが空の方を読んで除外することがある）。重複列は削除して1つにする
- タグが空 → AI処理日時が入っているか（未処理なら再分類）、値が英字コードか確認
- AIが `429 / RESOURCE_EXHAUSTED / limit: 0` → そのAPIキーに無料枠が無い状態。
  多くは大学・会社のGoogle Workspaceアカウントで無料枠が制限されているのが原因。
  **個人のGmailアカウントでAPIキーを作り直し**、`GEMINI_API_KEY` の値を差し替える。
  （スクリプトは大学アカウントのままでよい。鍵だけ個人アカウントのものにする）

## プライバシー面の注意

- 公開用シートに個人情報の列を作らない（本スクリプトは公開OK列だけを写します）
- 応募フォームに「**匿名プロフィールの公開に同意する**」チェック欄を用意しておく
